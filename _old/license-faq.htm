Aloha Editor GPL License FAQ

The following questions and answers are our personal interpretation and are not legally approved.

In short terms

You are free to

  • include Aloha Editor in GPLv2 or late licensed software and ship as one package
  • include Aloha Editor in MIT, BSD or similar licensed software and ship as one package with a FOSS license exception

You need to publish

  • the license to users
  • derived work to the users as "source code" (eg. as link in the comment of the site)

You may buy a commercial license (to support the community)

  • to support continuous development of Aloha Editor
  • if you want to sell a commercial software and don't want to publish derived work
  • if you want to provide any service using Aloha Editor and don't want to publish derived work

Our interpretation of following GPLv2 or later terms regarding Javascript code

  • "source code" are all single files as in the github repository.
  • "object code" are minified, combined or any with any other method modified "source code" files
  • "derived work" changes to the source code (except configs) of Aloha Editor

We invite you to check http://fi.am/entry/agplv3-explained/.

Frequently asked questions and answers

1. Aloha Editor is licensed "GPLv2 or later"? What does that mean?

It means that users will always be permitted to use it, and even change it, according to the terms of GPLv2 - even after later versions of the GPL (like GPLv3) are available. (for more detail see GNU FAQ)

2. I have server side software such as a CMS (or other software) GPLv3 licensed. Can I CHANGE OR EXTEND AND DISTRIBUTE Aloha Editor with it?

Yes, you can and we encourage you to do so. Aloha Editor is licensed under GPLv2 and later. That means later versions of the GPL (like GPLv3) are also valid.

3. I have server side software such as a CMS (or other software) BSD (revised), MIT, ASL or similar licensed. Can I CHANGE OR EXTEND AND DISTRIBUTE Aloha Editor with it?

Yes, you are allowed to to distribute Aloha Editor with BSD, MIT or similar licensed software after getting a FOSS license exception from us. You are not allowed to change the license of Aloha Editor itself. If you distribute derived work you need to publish the changes or the source code of the derived work GPLv2 or later compatible. Eg. Locomotive CMS is MIT, Aloha Editor is GPLv2 or later. With the FOSS exception, Locomotive CMS can distribute Aloha Editor and remain MIT, but Aloha Editor reamains GPLv2 or later. Contact us to get a FOSS license exception!

4. Do I need to publish modifications to a GPL server side component in every case?

No, the GPL does not require you to release your modified version, or any part of it in every case. You are free to make modifications and use them privately, without ever releasing them. This applies to organizations (including companies), too; an organization can make a modified version and use it internally without ever releasing it outside the organization. But if you release the modified version to the public in some way, the GPL requires you to make the modified source code available to the program's users, under the GPL. Publishing it to the Aloha Editor project is welcomed. See http://www.gnu.org/licenses/gpl-faq.html#GPLRequireSourcePostedPublic.

5. Do I need to publish extensions to a GPL server side component?

No, you do not need to publish extensions as not needed by the GPL from your server side software as long as you do not modify the software. If you modify it you may need to publish the changes depending on the GPL. See http://www.gnu.org/licenses/gpl-faq.html#GPLRequireSourcePostedPublic

6. I want to distribute Aloha Editor for a fee. Can I do that?

Yes, you can and we encourage you to do so. This is part of free software. See http://www.gnu.org/philosophy/selling.html.
